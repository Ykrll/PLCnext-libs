For acfprojects:

  -p, --path                  The path to the project settings file or the
                              project root directory. Default is the current
                              directory.

  -o, --output                The path where the files will be deployed in.
                              Default is the 'bin/target/Release' directory.
                              Relative paths are relative to the directory
                              defined with the '--path' option.

  -f, --files                 Additional files to be deployed. Files are
                              separated by ' '. Files need to be defined in the
                              following format:
                              path/to/source|path/to/destination(|target). The
                              path/to/source file will be relative to the root
                              folder, defined with the 'path' argument. It can
                              contain the wildcard '*' at the end of the path.
                              In this case all files from the directory and all
                              sub-directories will be deployed into the
                              destination directory without recreating the
                              directory structure. The path/to/destination will
                              be relative to the output/targetname/Release
                              directory, where output is defined with the
                              'output' argument. Optionally each file can have a
                              target definition. Without the target definition
                              the file is deployed to every target. Targets need
                              to be defined in the following format:
                              targetname(,targetversion). The version is
                              optional and is only needed if there are multiple
                              versions of a target in the project. The target
                              must match one of the targets of the project or
                              one of the targets defined with the 'target'
                              option, if it is defined.

  -t, --target                List of targets for which files are deployed.
                              Targets are separated by ' '. Targets need to be
                              defined in the following format:
                              targetname(,targetversion). The version is
                              optional and is only needed if there are multiple
                              versions of a target in the registered SDKs. If
                              used, this option overrides the targets defined in
                              the project. Please consider, that the actual
                              binaries are deployed in the build step. Therefore
                              if they are not built for a specific target,
                              neither are they built here.

  -b, --buildtype             Build type for which the deploy should be
                              executed. Default is 'Release'

  -i, --id                    The GUID for the library. If this option is not
                              used, the id stored in the project settings file
                              will be used instead.

  -d, --librarydescription    A description of the library. If this option is
                              used, a project configuration file will be
                              ignored. Use \n for linebreak and \" for quotation
                              mark

  -v, --libraryversion        The version of the library. If this option is
                              used, a project configuration file will be
                              ignored.

  -e, --engineerversion       The version of the PLCnext Engineer for which this
                              library shall be built.

  --solutionversion           The AdeSolutionVersion for which this library
                              shall be built. Can not be used together with
                              --engineerversion.

  --sign                      Set this option if the resulting library shall be
                              signed. A private key, public key and at least one
                              certificate need to be provided. They can be
                              combined in one PKCS#12 archive or provided as
                              separate PEM files. Set either --timestamp and
                              provide a timestamp server through
                              --timestampconfiguration to get a CAdES-T
                              signature or set '--notimestamp' to get CAdES-B.

  --pkcs12                    Path to PKCS#12 archive containing the private and
                              public key and certificates for signing.

  --privatekey                Path to PEM file containing the private key.

  --publickey                 Path to PEM file containing the public key.

  --certificates              Paths to PEM files containing certificates.
                              Separated by ','.

  --timestamp                 Set either this option to get a CAdES-T signature
                              or set '--notimestamp' to get CAdES-B.

  --notimestamp               Set either this option to get a CAdES-B signature
                              or set '--timestampserver' and provide
                              '--timestampconfiguration' to get CAdES-T.

  --password                  Password for private key file. If not set user
                              will be prompted to enter the password when
                              executing command.

  --timestampconfiguration    Path to timestamp configuration file (.json) for
                              setting e.g. timestampserver uri, proxy
                              settings... See info center for details.

  --verbose                   Enables verbose output.

  --quiet                     Suppresses all output.

  --help                      Display this help screen.

Examples:

Deploy files for all targets supported by project:
plcncli deploy --path Path/To/Project

Deploy doc/HelpText.txt to the doc directory in the deploy directory
additionally to the normally deployed files.:
plcncli deploy --path Path/To/Project --files doc/HelpText.txt|doc

Deploy library for targets AXCF2152 and RFC4072S:
plcncli deploy --path Path/To/Project --target AXCF2152 RFC4072S


For projects:

Deploy a .pcwlx library for 'PlmProject':
plcncli deploy --path PlmProject

Deploy 'PlmProject' with the custom library path 'lib/libPlmProject.so':
plcncli deploy --path PlmProject --files lib/libPlmProject.so|.

Deploy 'PlmProject' with the external libraries 'libA.so' and 'libB.so'.
Automatic external library detection is disabled with that:
plcncli deploy --path PlmProject --files extern/libA.so|. extern/libB.so|.

Deploy 'PlmProject' with the custom metadata path 'meta/device':
plcncli deploy --path PlmProject --files meta/device/*|.

Deploy 'PlmProject' into folder 'out' where binaries were built into standard
folder 'bin/AXCF2152_19.6.0.21013/Release':
plcncli deploy --path PlmProject --output out --files
bin/AXCF2152_19.6.0.21013/Release/*|.
